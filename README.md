# Daily Briefing Skill

定时汇报每日/每周任务日程，支持 iCal/ICS 订阅源，配置简单，开箱即用。

## 功能

自动从日历/任务管理工具（TickTick、Google Calendar、iCloud 等）的 iCal 订阅源读取数据，生成结构化简报：

| 汇报 | 时间 | 内容 |
|------|------|------|
| ☀️ 晨报 | 每天 8:00 | 今天的待办事项 |
| 🌃 晚报 | 每天 20:00 | 今天的事项汇总 |
| 📊 周报 | 每周日 22:00 | 本周事项汇总 |

## 特点

- **零依赖**：仅需 `web_fetch` + `cron`，无需额外工具
- **纯配置**：只需一个 iCal 链接，不需要 API 密钥或 OAuth
- **隐私安全**：只读订阅，不上传、不修改任何数据
- **灵活定制**：时间、渠道、语言均可自定义

## 快速部署

适用于任何支持 `web_fetch` 和 `cron` 工具的 OpenClaw 环境。

### 1. 获取 iCal 订阅链接

从你的日历/任务工具中获取 iCal 订阅 URL：

- **TickTick**：设置 → 日历订阅 → 复制链接（`webcal://` 改为 `https://`）
- **Google Calendar**：设置 → 集成日历 → 私享网址（iCal 格式）
- **Apple Calendar**：iCloud.com → 日历 → 共享日历

### 2. 安装技能

将 `SKILL.md` 放到你的 OpenClaw 技能目录：

```
skills/daily-briefing/SKILL.md
```

### 3. 配置参数

打开 SKILL.md，将以下 5 个占位符替换为你的实际值：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `<ICAL_URL>` | 你的 iCal 订阅链接 | `https://ticktick.com/...` |
| `<TIMEZONE>` | 时区 | `Asia/Shanghai` |
| `<DELIVERY_TO>` | 推送目标 | `telegram:123456789` |
| `<DELIVERY_CHANNEL>` | 推送渠道 | `telegram` |
| `<DELIVERY_ACCOUNT>` | 推送账号（可选） | `default` |

### 4. 创建定时任务

按照 SKILL.md 中的指引，用 `cron` 工具创建三个汇报任务。

### 5. 测试

创建后手动触发一次，确认输出格式：

```
cron action: "run"
jobId: <job-id>
runMode: "force"
```

## 输出示例

**晨报：**
```
☀️ 晨报 ｜ 5月24日 周六
- 10:00 项目评审会议
- 18:00 提交周报
```

**晚报：**
```
🌃 晚报 ｜ 5月24日 周六
- 10:00 项目评审会议（已安排）
- 18:00 提交周报（已安排）
```

**周报：**
```
📊 周报 ｜ 5月19日 - 5月25日
- 周一 14:00 部门周会
- 周三 09:00 客户演示
- 周五 17:00 代码提交截止
```

## 自定义

- **修改时间**：更改 cron 表达式，如工作日 7:30 → `"30 7 * * 1-5"`
- **禁用某个汇报**：设置 `"enabled": false`
- **修改语言**：翻译 `payload.message` 中的提示文本

## 注意事项

- iCal 是只读订阅，只能读取条目，无法判断完成状态
- 如果某时段没有条目，会简要说明，不写"无"、不留空模板
- 三个汇报的输出风格一致：标题 → 条目，无多余板块
