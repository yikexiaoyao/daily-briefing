# Daily Briefing Skill

定时汇报每日/每周任务日程，通过 dida CLI 直连滴答清单 API，支持优先级分组、已完成/未完成分类展示。

## 功能

通过 `dida` CLI（@suibiji/dida-cli）直连滴答清单 API，获取实时任务数据：

| 汇报 | 时间 | 内容 |
|------|------|------|
| ☀️ 晨报 | 每天 8:00 | 待办事项和未完成任务（按优先级分组） |
| 🌃 晚报 | 每天 20:00 | 今日未完成的待办事项 |
| 📊 周报 | 每周日 20:30 | 本周待跟进的未完成事项 |

## 特点

- **API 直连**：通过 dida CLI 获取实时数据，比 iCal 订阅更准确
- **优先级分组**：🔴 高 → 🟡 中 → 🟢 低 → 📋 无优先级
- **已完成/未完成分类**：清晰展示任务状态
- **跨天任务处理**：自动标注日期避免歧义（如"明天 08:00 出门带伞"）

## 前置条件

- 已安装 `@suibiji/dida-cli`（`npm i -g @suibiji/dida-cli`）
- 已执行 `dida auth login` 完成 OAuth 授权

## 快速部署

### 1. 安装 dida CLI

```bash
npm i -g @suibiji/dida-cli
```

### 2. 登录授权

```bash
dida auth login
```

### 3. 安装技能

将 `SKILL.md` 放到你的 OpenClaw 技能目录：

```
skills/daily-briefing/SKILL.md
```

### 4. 配置参数

打开 SKILL.md，将以下 3 个占位符替换为你的实际值：

| 占位符 | 说明 | 示例 |
|--------|------|------|
| `<DELIVERY_TO>` | 推送目标 | `telegram:123456789` |
| `<DELIVERY_CHANNEL>` | 推送渠道 | `telegram` |
| `<DELIVERY_ACCOUNT>` | 推送账号（可选） | `default` |

### 5. 创建定时任务

按照 SKILL.md 中的指引，用 `cron` 工具创建三个汇报任务。

### 6. 测试

创建后手动触发一次，确认输出格式：

```
cron action: "run"
jobId: <job-id>
runMode: "force"
```

## 跨天任务处理

当任务涉及跨天时，标题会自动包含日期以避免歧义：

- **当天任务**：`08:00 出门带伞`（19:30 提醒）
- **明天任务**：`明天 08:00 出门带伞`（今天 19:30 提醒）
- **后天任务**：`后天 14:00 会议`（今天提醒）

## 自定义

### 修改时间

修改 `expr` 字段即可：

| 场景 | Cron 表达式 |
|------|------------|
| 工作日 7:30 | `"30 7 * * 1-5"` |
| 每天两次（7:00 和 19:00） | `"0 7,19 * * *"` |
| 每 6 小时 | `"0 */6 * * *"` |
| 周一和周五 9:00 | `"0 9 * * 1,5"` |

### 禁用某个汇报

将 `"enabled": false` 即可暂停，无需删除。

### 修改语言

把 `payload.message` 中的中文提示翻译为其他语言即可。

## 与 v1.x 的区别

| 特性 | v1.x（iCal） | v2.0.0（dida CLI） |
|------|-------------|-------------------|
| 数据来源 | iCal 订阅（web_fetch） | dida CLI（API 直连） |
| 数据格式 | ICS 文本解析 | JSON |
| 支持功能 | 只读 | 完整 CRUD |
| 认证方式 | 公开链接 | OAuth token |
| 实时性 | 依赖订阅更新频率 | 实时 API 调用 |

## 注意事项

- **dida CLI 需要登录态**：token 存储在本地，过期需重新 `dida auth login`
- **不编造内容**：如果某时段没有条目，简要说明即可
- **格式统一**：三个汇报的输出风格一致
- **仅依赖 `dida` CLI + `cron`**：无需 iCal 订阅链接
